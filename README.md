# Trendsee Test Project

Репозиторий разделен на две независимые части:

- `backend/` — FastAPI + PostgreSQL + Redis
- `frontend/` — Vue 3 + Vite

## Что реализовано

### Backend

- создание пользователя с немедленной выдачей JWT;
- получение JWT по `user_id` для ручного тестирования;
- изменение и удаление пользователя;
- создание, изменение и удаление публикаций только для авторизованного пользователя;
- получение публикаций пользователя с кэшем в Redis на 10 минут;
- fallback в Postgres с искусственной задержкой `2s`, если записи нет в Redis;
- DI через `Depends` для сервисов, репозиториев и авторизации;
- параметризованные SQL-запросы без SQLAlchemy;
- демо-режим:
  - `GET /api/v1/users/demo` создает публичного demo-пользователя и стартовую ленту;
  - `POST /api/v1/posts/demo-seed` генерирует новые демо-публикации для авторизованного пользователя.

### Frontend

- pixel-close лента публикаций по макету;
- модалка подробного просмотра публикации;
- infinite scroll за `500px` до конца страницы;
- гостевой режим с публичной демо-лентой;
- регистрация через JWT прямо из интерфейса;
- сохранение сессии в `localStorage`;
- генерация дополнительных карточек кнопкой `Найти еще ролики`;
- сворачивание бокового меню с плавной анимацией;
- лайк на карточке с анимацией сердца;
- раскрытие `Creative +` в апсейл-блок `Улучшить подписку`.

## Архитектура backend

- `api` — роуты и HTTP-слой;
- `services` — бизнес-логика;
- `repositories` — доступ к PostgreSQL;
- `dependencies` — DI-провайдеры;
- `db` — подключение к PostgreSQL и Redis;
- `utils/demo_posts.py` — генерация фейкового контента для продукта.

## Запуск

Из корня проекта:

```bash
docker compose up --build
```

После запуска:

- frontend: `http://localhost:5173`
- backend: `http://localhost:8000`
- Swagger: `http://localhost:8000/docs`

## Основные эндпоинты

Префикс API: `/api/v1`

### Users

- `POST /users` — создать пользователя и получить JWT
- `GET /users/demo` — получить публичного demo-пользователя
- `GET /users/{user_id}/token` — получить токен по id
- `PATCH /users/{user_id}` — изменить имя
- `DELETE /users/{user_id}` — удалить пользователя

### Posts

- `POST /posts` — создать публикацию
- `PATCH /posts/{post_id}` — изменить публикацию
- `DELETE /posts/{post_id}` — удалить публикацию
- `GET /posts/user/{user_id}?limit=10&offset=0` — получить ленту пользователя
- `POST /posts/demo-seed?count=8&append=true` — сгенерировать демо-контент

## Идея продуктового сценария

Для тестового задания внешний video API не требуется. Источник карточек — это сами публикации в Postgres:

1. гость заходит на сайт и видит публичную демо-ленту;
2. после регистрации создается пользователь и выдается JWT;
3. backend генерирует для него стартовый набор демо-публикаций;
4. кнопка `Найти еще ролики` генерирует следующую пачку карточек;
5. frontend строит ленту и модалки только из данных API.

Такой сценарий делает проект законченным продуктовым прототипом без зависимости от внешних платформ.
